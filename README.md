# Idea Executor - 想法扩展与实现技能

当用户给出一个想法、需求或目标时，自动扩展和细化想法，分析可行性，补充必要细节，并逐步实现。

## 适用场景

- 项目规划（网站、应用、工具开发等）
- 需求分析（功能需求、业务需求）
- 代码实现（脚本、程序、自动化）
- 问题解决（性能优化、错误排查）
- 文档创建（方案、报告、说明文档）

## 工作流程

### 第零步：创建项目文档
在项目目录创建 `PROJECT.md` 文件，记录完整的方案、需求和进度。

### 第一步：读取历史经验
扫描 `1052/` 文件夹中的历史经验，为当前项目提供参考。

### 第二步：深度理解用户意图
- 识别想法类型（功能需求/问题解决/学习探索/项目规划/工具改进）
- 提取关键信息（Who/What/Why/Where/When/How）
- 与用户确认理解正确

### 第三步：想法扩展与分析
- 补全遗漏的功能需求
- 识别潜在问题并提供解决方案
- 提供专业建议和优化方案

### 第四步：制定详细执行计划
- 将项目分解为具体执行步骤
- 分析步骤间的依赖关系
- 评估风险并制定应对措施

### 第五步：逐步实现
- 按计划逐步执行
- 实时更新 PROJECT.md 进度
- 定期向用户汇报

### 第六步：交付与总结
- 完成交付检查
- 提供使用说明
- 询问是否沉淀经验到知识库

## 文件结构

```
idea-executor/
├── SKILL.md              # 技能核心文件
├── README.md             # 说明文档
└── 1052/                 # 经验沉淀文件夹
    ├── 2024-01-15_经验主题A.md
    ├── 2024-02-20_经验主题B.md
    └── ...
```

## PROJECT.md 结构

项目文档包含以下章节：

- **项目概述**：原始需求、需求类型、核心目标
- **需求分析**：关键要素、功能需求、非功能需求
- **可行性评估**：技术可行性、复杂度、风险
- **方案设计**：技术选型、架构设计
- **执行计划**：5个阶段的详细步骤
- **进度跟踪**：已完成、进行中、待处理
- **问题与记录**：遇到的问题、决策、变更
- **交付物**：完成的功能、使用说明

## 经验沉淀

项目完成后可选择将经验沉淀到 `1052/` 文件夹：

- **文件名格式**：`[日期]_[经验主题].md`
- **内容包含**：背景、核心经验、解决方案、适用场景、注意事项

### 经验文件示例

```markdown
# Python脚本自动化最佳实践

> 沉淀日期：2024-03-13
> 项目类型：工具开发

## 背景
用户需要一个自动备份文件的脚本

## 核心经验
### 1. 使用 argparse 处理命令行参数
比 input() 更适合自动化执行

### 2. 日志记录很重要
使用 logging 模块记录操作，便于排查问题

## 适用场景
- 文件备份
- 定时任务
- 自动化流程

## 注意事项
- 路径处理要兼容不同操作系统
```

## 使用方式

在 Claude 中使用此技能：

```
使用 idea-executor 技能帮我实现[你的想法]
```

例如：
- 「使用 idea-executor 技能帮我做一个个人博客」
- 「帮我写一个自动备份文件的脚本，用 idea-executor 技能」
